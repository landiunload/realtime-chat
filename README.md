# Realtime Chat

Чат реального времени на **ASP.NET Core SignalR** с сохранением истории в **PostgreSQL**.
Код организован по принципу **Vertical Slice Architecture** — папки соответствуют возможностям системы, а не техническим слоям.

## Стек технологий

- **.NET 9 / ASP.NET Core SignalR** — двусторонняя связь по WebSocket с автоматическим fallback
- **Entity Framework Core 9 + PostgreSQL** — хранение истории сообщений
- **Serilog** — структурированное логирование
- **xUnit + NSubstitute** — модульные тесты
- **Docker Compose** — локальное окружение

## Ключевые решения

- **Строго типизированный хаб** `Hub<IChatClient>`: контракт клиентских методов описан
  интерфейсом, поэтому опечатка в имени метода — ошибка компиляции, а не молчаливый сбой.
- **Комнаты на группах SignalR**: сообщения доставляются только участникам нужной комнаты.
- **История при входе**: новый участник сразу получает последние 50 сообщений комнаты.
- **Хранилище за интерфейсом** `IChatMessageStore` (инверсия зависимостей):
  PostgreSQL-реализация подменяется в тестах без поднятия базы.
- **Инварианты в модели**: сообщение создаётся только фабричным методом с проверками,
  составной индекс `(RoomName, SentAtUtc)` оптимизирует основной сценарий чтения.

## Запуск

```bash
docker compose up --build
```

Открой `http://localhost:8081` в двух вкладках браузера, войди в одну комнату
под разными именами — сообщения доставляются мгновенно, история сохраняется.
